tkDashboard 接口文档

基于三个前端页面的交互与弹框设计，约定后端接口格式。对应页面：
销售数据映射/htmls/店铺统计.html、销售数据映射/htmls/商品统计.html、PID/htmls/需求二-数据看板.html。

通用约定

项说明
Base URL待定（如 /api/bi/tk）
请求格式GET 使用 query string；POST 使用 application/json
响应格式application/json（导出为 text/csv 或文件流）
分页page（从 1 开始）、pageSize（默认 20）；响应含 total
时间格式日期 YYYY-MM-DD；日期时间 YYYY-MM-DD HH:mm:ss
鉴权待定（Cookie / Token）

统一响应结构

{
  "code": 0,
  "message": "",
  "data": { }
}

code !== 0 表示业务错误，data 为接口约定结构。

零、下拉选项接口（三个页面共用）

店铺统计、商品统计、PID 数据看板三个页面均有「站点」级联选择与「店铺」下拉，店铺可根据站点（区域/国家）筛选。以下接口供前端初始化或切换时调用。

一、需求一：每日订单数播报（店铺统计）

对应页面：店铺统计.html。无弹框。

1.1 查询条件

  • 指标：销量（qty）/ 销售额（amount）
  • 站点：级联（区域 → 国家），仅用于前端筛选店铺列表，主表接口不传 region/country
  • 店铺：单选，空=全部
  • 日期范围：开始日期、结束日期
  • 日期类型：日 / 周 / 月 / 季 / 年（决定表头 bucket 与环比对比基准）
  • 销售额时：可选最低金额筛选、统一展示币种

1.2 主表接口：店铺统计列表

实现类：TkStoreStatsController#getStoreSiteStats，数据来源：Doris ads_sales_store_site_order_count_realtime。

Content-Type: application/json

请求参数说明

参数类型必填说明

data 字段说明（按层级）

层级路径类型说明
1storeNamestring当前筛选范围：传 storeName 时为该值，未传时为「全部」
1startDate / endDatestring查询日期范围，YYYY-MM-DD
1granstring日期粒度，决定 bucket 与环比基准
1metricstringqty 订单数 / amount 实付金额
1displayCurrencystring销售额展示币种；未传或 original 时 list 各行用原币
1bucketslist时间 bucket 列表，与表头列一致，新→旧
2buckets[].labelstring展示标签，如「2026/2/28」
1listlist各店铺站点统计行，每行对应一个 store_site
2list[].store_site_namestring店铺站点名称（store_site_name 优先，空则 store_name，再空则 store_code|store_site_code）
2list[].currencystring该行金额币种；amount 模式且原币时为站点 currency_code；统一币种时为 displayCurrency；合计行多币种时为「—」
2list[].totalnumber/string汇总值；minAmount 按日过滤下无达标日时为 --
2list[].ratiostring占合计百分比，如「45.20%」
2list[].avgnumber/string各 bucket 值的算术平均；无有效数值时为 --
2list[].chainRatiostring环比：首 bucket 与上一周期对比，如「+12.5%」
2list[].valslist各 bucket 对应值，与 buckets 顺序一致；无达标日时为 --
1totalRowobject合计行，结构同 list 单行
2totalRow.store_site_namestring固定「合计」
2totalRow.ratiostring固定「100%」或「—」（全表 -- 时）

业务逻辑梳理

  1. 参数校验：startDate、endDate 必填且格式 YYYY-MM-DD；startDate ≤ endDate；metric 仅支持 qty 或 amount。
  2. 数据来源：查询 Doris ads.ads_sales_store_site_order_count_realtime，字段含 sdt、store_code、store_site_code、store_name、store_site_name、order_count、valid_order_count、paid_amount、currency_code（及 paid_amount_rmb 用于 minAmount 人民币比较）。
  3. 店铺筛选：storeName 非空时按 store_site_name 精确匹配；为空时若配置了 tk.jg.sync.storeSiteNames 则限定在该列表，否则查全部。
  4. 指标计算：metric=qty 时取 order_count 汇总；metric=amount 时取 paid_amount。按 gran 将 sdt 映射到 bucket（day→yyyyMMdd，week→周首日 yyyyMMdd，month→yyyyMM，quarter→yyyyQn，year→yyyy），同一 bucket 内聚合。
  5. minAmount 过滤（仅 amount 模式）：按自然日比较。原币：COALESCE(paid_amount,0) >= minAmount；指定展示币种：先将 minAmount 换算为人民币，与 paid_amount_rmb 比较。某日不达标则该日明细不参与该 bucket 汇总；整行无达标日时 total/avg/vals 为 --。
  6. 展示币种换算（amount 且 displayCurrency 非空）：从 dim.dim_exchange_rate_df 取 USD→CNY 汇率，再按配置换算各币种→展示币种。每行金额统一换算后展示。
  7. 环比：首 bucket 与上一周期（按 gran 推算）对比，计算百分比，格式如 +12.5%、-3.2%。
  8. 占比：每行 total / 合计 total × 100%，保留两位小数。

1.3 导出

POST /tk/sales/storeSiteStats/export
Content-Type: application/json

请求体同 1.2，返回 Excel（storeSiteStats.xlsx）。表头：店铺、币种、小计、占比、平均值、环比、各 bucket 列（与 data.buckets 顺序一致）。

三、需求二：PID 数据看板

对应页面：需求二-数据看板.html。主表支持对比期；弹框为 PID 明细（与 2.4 同一接口）。

3.1 查询条件

  • 站点：级联（区域 → 国家），仅用于前端筛选店铺列表，主表接口不传 region/country
  • 店铺：单选，空=全部
  • 日期范围：开始日期、结束日期
  • 对比时间：可选，用于主表环比（当期 vs 对比期）；传 compareStart、compareEnd
  • 搜索：字段（PID / SPU）+ 关键词，模糊

3.2 主表接口：数据看板列表

GET /api/bi/tk/pid/dashboard

查询参数

参数类型必填说明
startDatestring否开始日期 YYYY-MM-DD
endDatestring否结束日期 YYYY-MM-DD
storeNamestring否店铺名称，空=全部
searchFieldstring否pid / spu，搜索维度
searchKeywordstring否模糊搜索关键词

参与对比的字段

除 date、pid、spu、分析 外的 47 个数据列统一以数组返回，结构一致：[当前值, 对比值, 环比百分比]。未开启对比（未传 compareStart/compareEnd）时，第二、第三项为空（null 或 ""）；开启对比时，对比值、环比由后端计算并填入，环比格式如 "+12.5%"、"-3.2%"，无法计算时可返回 null 或 "—"。

参与对比的字段（序号 5～51）：atv, subsidyPer, financialGmv, tradeAmount, dealQty, orderQty, cardGmv, liveGmv, videoGmv, cardGmvRatio, liveGmvRatio, videoGmvRatio, cardTradeAmount, cardDealQty, cardAtv, cardExposure, cardPageView, cardUvView, cardUvUser, cardClick, cardConvert, cardVideoTradeAmount, videoDealQty, videoAtv, videoExposure, videoPageView, videoUvView, videoUvUser, videoClickRate, videoConvertRate, bizHangerQty, editHangerQty, totalHangerQty, bizOpenVideoQty, editOpenVideoQty, totalOpenVideoQty, vasConsume, gmvmaxConsume, adTotalConsume, adOrderCount, adOrderQty, adGmv, totalGmv, totalAcoas, adRoi, adAcoas, adRoiSettingCount。

响应 data

上述 47 个字段始终为三元数组 [当前值, 对比值, 环比百分比]；未开启对比时对比值、环比为空。

查询参数

参数类型必填说明
pidstring是PID
storeNamestring是店铺名称

响应 data

3.4 弹框：PID 明细

与 2.4 相同，使用 GET /api/bi/tk/pid/detail。数据看板主表点击某行「分析」时，固定该行的 SPU（及店铺），打开弹框后用户可在弹框内再筛选：选择时间范围（startDate、endDate）、日期维度（gran：日/周/月），每次变更后前端带当前弹框参数重新请求接口，刷新 51 列明细表与合计行。参数：spu、storeName（来自点击行）、startDate、endDate、gran（来自弹框内选择）。

3.5 导出

GET /api/bi/tk/pid/dashboard/export

查询参数同 3.2（不含 page、pageSize），返回 CSV，列与主表一致。

#页面功能Method路径

4需求一 店铺统计主表POST/tk/sales/storeSiteStats
5需求一 店铺统计导出POST/tk/sales/storeSiteStats/export
6需求二 商品统计主表GET/api/bi/tk/sales/product-stats

9需求二 商品统计导出GET/api/bi/tk/sales/product-stats/export
10需求三 数据看板主表GET/api/bi/tk/pid/dashboard

  • PID：全称 Product ID（产品 ID），对应 TikTok API 订单明细中的 product_id 字段。
