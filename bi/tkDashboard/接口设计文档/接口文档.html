tkDashboard 接口文档

基于三个前端页面的交互与弹框设计，约定后端接口格式。对应页面：
销售数据映射/htmls/店铺统计.html、销售数据映射/htmls/商品统计.html、PID/htmls/需求二-数据看板.html。

通用约定

项说明
Base URL待定（如 /api/bi/tk）
请求格式GET 使用 query string；POST 使用 application/json
响应格式application/json（导出为 text/csv 或文件流）
分页page（从 1 开始）、pageSize（默认 20）；响应含 total
时间格式日期 YYYY-MM-DD；日期时间 YYYY-MM-DD HH:mm:ss
鉴权待定（Cookie / Token）

统一响应结构

{
  "code": 0,
  "message": "",
  "data": { }
}

code !== 0 表示业务错误，data 为接口约定结构。

一、需求一：每日订单数播报（店铺统计）

对应页面：店铺统计.html。无弹框。

1.1 查询条件

  • 指标：销量（qty）/ 销售额（amount）
  • 站点：级联（区域 → 国家），仅用于前端筛选店铺列表，主表接口不传 region/country
  • 店铺：单选，空=全部
  • 日期范围：开始日期、结束日期
  • 日期类型：日 / 周 / 月 / 季 / 年（决定表头 bucket 与环比对比基准）
  • 销售额时：可选最低金额筛选、统一展示币种

1.2 主表接口：店铺统计列表

GET /api/bi/tk/sales/store-stats

查询参数

参数类型必填说明
metricstring否qty 销量 / amount 销售额，默认 qty
storeNamestring否店铺名称，空=全部
startDatestring是开始日期 YYYY-MM-DD
endDatestring是结束日期 YYYY-MM-DD
granstring否日期类型：day / week / month / quarter / year，默认 day
minAmountnumber否销售额模式下最低金额过滤
displayCurrencystring否销售额统一展示币种（如 USD、EUR），空为原始币种

data 字段说明（按层级）

层级路径类型说明
1storeNamestring当前筛选的店铺范围：单选店铺时为该店铺名，未选时为「全部」
1startDatestring查询开始日期，格式 YYYY-MM-DD，与请求参数一致
1endDatestring查询结束日期，格式 YYYY-MM-DD，与请求参数一致
1granstring日期粒度：day / week / month / quarter / year，决定表头列与环比对比基准
1metricstring指标类型：qty 销量 / amount 销售额
1bucketslist与表头列一致，按 gran 聚合后的时间 bucket 列表，顺序与前端表头一致
2buckets[].labelstring单个 bucket 的展示标签，如「2026/2/28」，用于表头展示
1listlist各店铺统计行，每行对应一个店铺，顺序与前端表格一致
2list[].storeNamestring店铺名称
2list[].totalnumber该店铺在 startDate～endDate 内的汇总值（销量为整数，销售额为小数）
2list[].ratiostring该店铺小计占合计的百分比，如「45.20%」
2list[].avgnumber各 bucket 值的算术平均（销量取整，销售额保留小数）
2list[].chainRatiostring环比：当前期（首 bucket）与上一周期对比的百分比，如「+12.5%」「-3.2%」
2list[].valslist各 bucket 对应的值，与 buckets 顺序、长度一致；销量为 number，销售额为 number
1totalRowobject合计行，与 list 中每行结构一致，表示全表合计
2totalRow.storeNamestring固定为「合计」
2totalRow.totalnumber所有店铺的汇总值
2totalRow.ratiostring固定为「100%」
2totalRow.avgnumber合计行的平均值（按 bucket 数）
2totalRow.chainRatiostring合计行的环比百分比
2totalRow.valslist各 bucket 的合计值，与 buckets 顺序、长度一致

1.3 导出

GET /api/bi/tk/sales/store-stats/export

查询参数同 1.2，返回 CSV：店铺、小计、占比、平均值、环比、各 bucket 列。

二、需求二：订单明细与销量趋势（商品统计）

对应页面：商品统计.html。含两个弹框：AI 分析、PID 明细。

2.1 查询条件

  • 站点：级联 → 店铺列表
  • 店铺：必选
  • 搜索：字段（SPU / PID）+ 关键词，模糊
  • 日期范围：开始、结束
  • 日期类型：日 / 周 / 月 / 季 / 年

2.3 弹框接口：AI 分析（按 SPU）

用于商品统计页「AI」按钮，弹框内展示该 SPU 的文本摘要。

POST /api/bi/tk/sales/product-stats/ai-analysis

请求 Body

{
  "storeName": "美国 TK・艾斯特尼 - 美区特 1 店 US",
  "spu": "AL-W0161",
  "startDate": "2026-02-21",
  "endDate": "2026-02-27",
  "gran": "day"
}

响应 data：spu、analysis（文本摘要）。

2.4 弹框接口：PID 明细（商品统计 / 数据看板共用）

用于商品统计页「分析」、需求三数据看板「分析」按钮。弹框内固定主推 SPU（数据看板场景下为当前行的 SPU，对应行仅一个 PID 时可视为固定 PID），用户可在弹框内再筛选以刷新明细表。

弹框界面要点（对应 需求二-数据看板.html 内 PID 明细弹框）：

  • 标题：PID 明细
  • 筛选区：时间范围（开始日期、结束日期）、日期维度（日 / 周 / 月）；修改后前端带新参数重新请求本接口刷新表格
  • 表格：与主表一致的 51 列表头（日期、PID、主推SPU、分析、客单价、…、广告ROI设置数），支持合计行；按 gran 展示按日或按周/月聚合后的明细
GET /api/bi/tk/pid/detail

查询参数：spu（弹框固定，来自点击行）、storeName（弹框固定）、startDate、endDate（弹框内选择）、gran（day / week / month，弹框内切换后带新值请求）。

响应 data：spu、storeName、startDate、endDate、gran、headers（51 列）、rows、totalRow。与需求文档「数据看板」映射一致。

2.5 导出（商品统计）

GET /api/bi/tk/sales/product-stats/export

查询参数同 2.2，返回 CSV：店铺名、SPU(公式)、PID、各 bucket 列、环比。

三、需求二：PID 数据看板

对应页面：需求二-数据看板.html。主表支持对比期；弹框为 PID 明细（与 2.4 同一接口）。

3.1 查询条件

  • 站点：级联（区域 → 国家），仅用于前端筛选店铺列表，主表接口不传 region/country
  • 店铺：单选，空=全部
  • 日期范围：开始日期、结束日期
  • 对比时间：可选，用于主表环比（当期 vs 对比期）；传 compareStart、compareEnd
  • 搜索：字段（PID / SPU）+ 关键词，模糊

3.3 每日明细接口（点击「每日」按钮）

对应页面：需求二-数据看板.html。主表每行 PID 旁有「每日」按钮，点击后在该行下方展开该 PID 在当前查询日期范围内的按日明细表（表头与主表一致，每行一天）。

GET /api/bi/tk/pid/dashboard/daily

查询参数

参数类型必填说明
pidstring是PID
storeNamestring是店铺名称
startDatestring是开始日期 YYYY-MM-DD（与主表查询条件一致）
endDatestring是结束日期 YYYY-MM-DD

响应 data

返回该 PID + 店铺在 startDate～endDate 内按日的明细列表，结构与 3.2 主表 list[] 单条一致（51 列字段），无分页；按日期正序。字段：pid、storeName、startDate、endDate、list（每项含 date、pid、spu 及与主表相同的 51 列字段，可不含 _compare）。前端用 list 在「每日」展开区域渲染与主表同表头的表格，一行一天。

3.4 弹框：PID 明细

与 2.4 相同，使用 GET /api/bi/tk/pid/detail。数据看板主表点击某行「分析」时，固定该行的 SPU（及店铺），打开弹框后用户可在弹框内再筛选：选择时间范围（startDate、endDate）、日期维度（gran：日/周/月），每次变更后前端带当前弹框参数重新请求接口，刷新 51 列明细表与合计行。参数：spu、storeName（来自点击行）、startDate、endDate、gran（来自弹框内选择）。

3.5 导出

GET /api/bi/tk/pid/dashboard/export

查询参数同 3.2（不含 page、pageSize），返回 CSV，列与主表一致。
