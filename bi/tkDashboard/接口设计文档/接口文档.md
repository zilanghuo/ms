# tkDashboard 接口文档

基于三个前端页面的交互与弹框设计，约定后端接口格式。对应页面：  
`销售数据映射/htmls/店铺统计.html`、`销售数据映射/htmls/商品统计.html`、`PID/htmls/需求二-数据看板.html`。

---

## 通用约定

| 项 | 说明 |
|---|---|
| Base URL | 待定（如 `/api/bi/tk`） |
| 请求格式 | GET 使用 query string；POST 使用 `application/json` |
| 响应格式 | `application/json`（导出为 `text/csv` 或文件流） |
| 分页 | `page`（从 1 开始）、`pageSize`（默认 20）；响应含 `total` |
| 时间格式 | 日期 `YYYY-MM-DD`；日期时间 `YYYY-MM-DD HH:mm:ss` |
| 鉴权 | 待定（Cookie / Token） |

**统一响应结构**

```json
{
  "code": 0,
  "message": "",
  "data": { }
}
```

`code !== 0` 表示业务错误，`data` 为接口约定结构。

## 一、需求一：每日订单数播报（店铺统计）

对应页面：`店铺统计.html`。无弹框。

### 1.1 查询条件

- **指标**：销量（qty）/ 销售额（amount）
- **站点**：级联（区域 → 国家），仅用于前端筛选店铺列表，主表接口不传 region/country
- **店铺**：单选，空=全部
- **日期范围**：开始日期、结束日期
- **日期类型**：日 / 周 / 月 / 季 / 年（决定表头 bucket 与环比对比基准）
- **销售额时**：可选最低金额筛选、统一展示币种

### 1.2 主表接口：店铺统计列表

```
GET /api/bi/tk/sales/store-stats
```

**查询参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| metric | string | 否 | `qty` 销量 / `amount` 销售额，默认 `qty` |
| storeName | string | 否 | 店铺名称，空=全部 |
| startDate | string | 是 | 开始日期 YYYY-MM-DD |
| endDate | string | 是 | 结束日期 YYYY-MM-DD |
| gran | string | 否 | 日期类型：day / week / month / quarter / year，默认 day |
| minAmount | number | 否 | 销售额模式下最低金额过滤 |
| displayCurrency | string | 否 | 销售额统一展示币种（如 USD、EUR），空为原始币种 |

**响应 data**

```json
{
  "storeName": "全部",
  "startDate": "2026-02-01",
  "endDate": "2026-02-28",
  "gran": "day",
  "metric": "qty",
  "buckets": [
    { "label": "2026/2/28" },
    { "label": "2026/2/27" },
    { "label": "2026/2/26" },
    { "label": "2026/2/25" },
    { "label": "2026/2/24" }
  ],
  "list": [
    {
      "storeName": "美国 TK・艾斯特尼 - 美区特 1 店 US",
      "total": 1250,
      "ratio": "45.20%",
      "avg": 44,
      "chainRatio": "+12.5%",
      "vals": [38, 42, 45, 41, 44]
    }
  ],
  "totalRow": {
    "storeName": "合计",
    "total": 2760,
    "ratio": "100%",
    "avg": 98,
    "chainRatio": "+8.2%",
    "vals": [90, 95, 92, 88, 98]
  }
}
```

**data 字段说明（按层级）**

| 层级 | 路径 | 类型 | 说明 |
|------|------|------|------|
| 1 | storeName | string | 当前筛选的店铺范围：单选店铺时为该店铺名，未选时为「全部」 |
| 1 | startDate | string | 查询开始日期，格式 YYYY-MM-DD，与请求参数一致 |
| 1 | endDate | string | 查询结束日期，格式 YYYY-MM-DD，与请求参数一致 |
| 1 | gran | string | 日期粒度：day / week / month / quarter / year，决定表头列与环比对比基准 |
| 1 | metric | string | 指标类型：qty 销量 / amount 销售额 |
| 1 | buckets | list | 与表头列一致，按 gran 聚合后的时间 bucket 列表，顺序与前端表头一致 |
| 2 | buckets[].label | string | 单个 bucket 的展示标签，如「2026/2/28」，用于表头展示 |
| 1 | list | list | 各店铺统计行，每行对应一个店铺，顺序与前端表格一致 |
| 2 | list[].storeName | string | 店铺名称 |
| 2 | list[].total | number | 该店铺在 startDate～endDate 内的汇总值（销量为整数，销售额为小数） |
| 2 | list[].ratio | string | 该店铺小计占合计的百分比，如「45.20%」 |
| 2 | list[].avg | number | 各 bucket 值的算术平均（销量取整，销售额保留小数） |
| 2 | list[].chainRatio | string | 环比：当前期（首 bucket）与上一周期对比的百分比，如「+12.5%」「-3.2%」 |
| 2 | list[].vals | list | 各 bucket 对应的值，与 buckets 顺序、长度一致；销量为 number，销售额为 number |
| 1 | totalRow | object | 合计行，与 list 中每行结构一致，表示全表合计 |
| 2 | totalRow.storeName | string | 固定为「合计」 |
| 2 | totalRow.total | number | 所有店铺的汇总值 |
| 2 | totalRow.ratio | string | 固定为「100%」 |
| 2 | totalRow.avg | number | 合计行的平均值（按 bucket 数） |
| 2 | totalRow.chainRatio | string | 合计行的环比百分比 |
| 2 | totalRow.vals | list | 各 bucket 的合计值，与 buckets 顺序、长度一致 |

### 1.3 导出

```
GET /api/bi/tk/sales/store-stats/export
```

查询参数同 1.2，返回 CSV：店铺、小计、占比、平均值、环比、各 bucket 列。

## 二、需求二：订单明细与销量趋势（商品统计）

对应页面：`商品统计.html`。含两个弹框：**AI 分析**、**PID 明细**。

### 2.1 查询条件

- **站点**：级联 → 店铺列表
- **店铺**：必选
- **搜索**：字段（SPU / PID）+ 关键词，模糊
- **日期范围**：开始、结束
- **日期类型**：日 / 周 / 月 / 季 / 年

GET /api/bi/tk/sales/product-stats

| endDate | string | 是 | 结束日期 YYYY-MM-DD |
| gran | string | 否 | day / week / month / quarter / year，默认 day |
| searchField | string | 否 | spu / pid |
| searchKeyword | string | 否 | 模糊搜索关键词 |

### 2.3 弹框接口：AI 分析（按 SPU）

用于商品统计页「AI」按钮，弹框内展示该 SPU 的文本摘要。

```
POST /api/bi/tk/sales/product-stats/ai-analysis
```

**请求 Body**

```json
{
  "storeName": "美国 TK・艾斯特尼 - 美区特 1 店 US",
  "spu": "AL-W0161",
  "startDate": "2026-02-21",
  "endDate": "2026-02-27",
  "gran": "day"
}
```

**响应 data**

```json
{
  "spu": "AL-W0161",
  "analysis": "【美国 TK・艾斯特尼 - 美区特 1 店 US · AL-W0161】上期 合计 95 件，本期 合计 82 件，环比下降；环比 -13.5%。\n\n共 4 个 PID。\n\n环比为负的 PID：…67205（-2.4%）、…78149（-5.1%），建议关注库存或推广。\n\n以上为基于该 SPU 数据的结构化摘要，可配合运营做补货或复盘。"
}
```

### 2.4 弹框接口：PID 明细（商品统计 / 数据看板共用）

用于商品统计页「分析」按钮、以及需求三数据看板「分析」按钮。弹框内**固定主推 SPU**（数据看板场景下为当前行的 SPU，对应行仅一个 PID 时可视为固定 PID），用户可在弹框内**再筛选**以刷新明细表。

**弹框界面要点**（对应 `需求二-数据看板.html` 内 PID 明细弹框）：

- **标题**：PID 明细
- **筛选区**：时间范围（开始日期、结束日期）、日期维度（日 / 周 / 月）；修改后前端带新参数重新请求本接口刷新表格
- **表格**：与主表一致的 51 列表头（日期、PID、主推SPU、分析、客单价、…、广告ROI设置数），支持合计行；按 `gran` 展示按日或按周/月聚合后的明细

```
GET /api/bi/tk/pid/detail
```

**查询参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| spu | string | 是 | 主推 SPU（弹框固定，来自点击行） |
| storeName | string | 是 | 店铺名称（弹框固定，来自主表或当前筛选） |
| startDate | string | 是 | 弹框内选择的开始日期 |
| endDate | string | 是 | 弹框内选择的结束日期 |
| gran | string | 否 | day / week / month，默认 day，弹框内切换后带新值请求 |

**响应 data**

```json
{
  "spu": "AL-W0161",
  "storeName": "美国 TK・艾斯特尼 - 美区特 1 店 US",
  "startDate": "2025-12-10",
  "endDate": "2026-03-12",
  "gran": "day",
  "headers": ["日期", "PID", "主推SPU", "分析", "客单价", "平均补贴/单", "财务GMV", "…"],
  "rows": [
    ["2026/2/27", "1729439482671567205", "AL-W0161", "—", 68, 3.82, 28500, "…"]
  ],
  "totalRow": ["合计", "", "", "", 68, 3.80, 85200, "…"]
}
```

- `headers`：51 列表头，与需求文档「数据看板」映射一致（日期、PID、主推SPU、分析、客单价、平均补贴/单、财务GMV、商品交易总额4、…、广告ROI设置数）。
- `rows`：每行 51 个单元格，可为 number / string（含百分比如 `"12.5%"`）。
- `totalRow`：合计行，51 列；占比/率列可为空或 `""`。

### 2.5 导出（商品统计）

```
GET /api/bi/tk/sales/product-stats/export
```

查询参数同 2.2，返回 CSV：店铺名、SPU(公式)、PID、各 bucket 列、环比。

## 三、需求二：PID 数据看板

对应页面：`需求二-数据看板.html`。主表支持对比期；弹框为 **PID 明细**（与 2.4 同一接口）。

- **站点**：级联（区域 → 国家），仅用于前端筛选店铺列表，主表接口不传 region/country
- **店铺**：单选，空=全部
- **日期范围**：开始日期、结束日期
- **对比时间**：可选，用于主表环比（当期 vs 对比期）；传 compareStart、compareEnd
- **搜索**：字段（PID / SPU）+ 关键词，模糊

### 3.3 每日明细接口（点击「每日」按钮）

对应页面：`需求二-数据看板.html`。主表每行 PID 旁有「每日」按钮，点击后在该行下方展开该 PID 在**当前查询日期范围内**的按日明细表（表头与主表一致，每行一天）。

```
GET /api/bi/tk/pid/dashboard/daily
```

**查询参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| pid | string | 是 | PID |
| storeName | string | 是 | 店铺名称 |
| startDate | string | 是 | 开始日期 YYYY-MM-DD（与主表查询条件一致） |
| endDate | string | 是 | 结束日期 YYYY-MM-DD |

**响应 data**

返回该 PID + 店铺在 startDate～endDate 内**按日**的明细列表，结构与 3.2 主表 `list[]` 单条一致（51 列字段），无分页；按日期正序。

### 3.4 弹框：PID 明细

与 **2.4** 相同，使用 `GET /api/bi/tk/pid/detail`。数据看板主表点击某行「分析」时，**固定该行的 SPU（及店铺）**，打开弹框后用户可在弹框内**再筛选**：选择时间范围（startDate、endDate）、日期维度（gran：日/周/月），每次变更后前端带当前弹框参数重新请求接口，刷新 51 列明细表与合计行。参数：`spu`、`storeName`（来自点击行）、`startDate`、`endDate`、`gran`（来自弹框内选择）。

### 3.5 导出

```
GET /api/bi/tk/pid/dashboard/export
```

查询参数同 3.2（不含 page、pageSize），返回 CSV，列与主表一致。
