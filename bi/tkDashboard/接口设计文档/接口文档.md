# tkDashboard 接口文档

基于 5 个前端 Demo HTML 页面反推的后端接口设计，覆盖查询、分页、导出与 AI 分析。

---

## 通用约定

| 项 | 说明 |
|---|---|
| Base URL | `待定`（如 `/api/bi/tk` ） |
| 请求格式 | `application/json`；GET 用 query string |
| 响应格式 | `application/json`（导出接口返回 `text/csv`） |
| 分页 | `page`（从 1 开始）、`pageSize`（默认 10）；响应含 `total` |
| 时间格式 | 日期 `YYYY-MM-DD`；日期时间 `YYYY-MM-DD HH:mm:ss` |
| 鉴权 | 待定（Cookie / Token） |

## 一、PID 需求一：GMV 计算表

### 1.1 查询列表

```
GET /api/bi/tk/pid/gmv
```

**查询参数**

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| date | string | 否 | 日期，格式 `YYYY-MM-DD` |
| store | string | 否 | 店铺编码（如 `1店`、`2店`、`3店`、`GB`），空=全部 |
| pid | string | 否 | PID，支持模糊搜索（包含匹配） |
| page | int | 否 | 页码，默认 1 |
| pageSize | int | 否 | 每页条数，默认 10 |

**响应 Body**

```json
{
  "code": 0,
  "data": {
    "total": 6,
    "page": 1,
    "pageSize": 10,
    "list": [
      {
        "date": "2026-02-23",
        "pid": "1729773126172578149",
        "aux": "2026/2/231729773126172578149",
        "revenue": 11298.87,
        "discount": 4909.31,
        "shipping": 352.47,
        "gmv": 6742.03,
        "qty": 113,
        "subsidy": 527.80,
        "subsidyPer": 4.67,
        "store": "1店",
        "paid": 5861.76,
        "atv": 51.87
      }
    ]
  }
}
```

**字段说明**

| 字段 | 类型 | 说明 |
|---|---|---|
| date | string | 日期 |
| pid | string | PID |
| aux | string | 辅助列（日期+PID拼接） |
| revenue | number | 营业总额 |
| discount | number | 商家折扣 |
| shipping | number | 运费 |
| gmv | number | 总GMV |
| qty | int | 单量 |
| subsidy | number | 补贴 |
| subsidyPer | number | 每单补贴 |
| store | string | 店铺 |
| paid | number | 实付总价（折扣价-补贴+运费） |
| atv | number | 客单价（实付总价/单量） |

### 1.2 导出 CSV

```
GET /api/bi/tk/pid/gmv/export
```

查询参数同 1.1（不含分页），返回 `text/csv` 文件流，字段顺序与表格一致。

## 三、销售数据映射 需求一：每日订单数播报

### 3.1 查询列表

```
GET /api/bi/tk/sales/daily-orders
```

**查询参数**

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| date | string | 否 | 日期，格式 `YYYY-MM-DD`，空=全部 |

**响应 Body**

```json
{
  "code": 0,
  "data": {
    "list": [
      {
        "dateLabel": "2月27日（美国时间）",
        "date": "2026-02-27",
        "store": "1店（美区跨境1店）",
        "orders": 230,
        "isTotal": false
      },
      {
        "dateLabel": "2月27日（美国时间）",
        "date": "2026-02-27",
        "store": "合计",
        "orders": 245,
        "isTotal": true
      }
    ]
  }
}
```

**字段说明**

| 字段 | 类型 | 说明 |
|---|---|---|
| dateLabel | string | 日期展示文案（如 `2月27日（美国时间）`） |
| date | string | 日期 YYYY-MM-DD |
| store | string | 店铺名称（含「合计」行） |
| orders | int | 订单数 |
| isTotal | bool | 是否合计行 |

### 3.2 导出 CSV

```
GET /api/bi/tk/sales/daily-orders/export
```

查询参数同 3.1，返回 `text/csv`，字段：日期、店铺、订单数。

## 四、销售数据映射 需求二：订单明细与销量趋势

### 4.1 查询销量趋势（SPU→PID 聚合）

```
GET /api/bi/tk/sales/trend
```

**查询参数**

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| store | string | 是 | 店铺编码（`1店`、`2店`、`3店`、`英区`） |
| endDate | string | 否 | 到日期，格式 `YYYY-MM-DD`；表格最右列为该日，共展示该日往前 7 天；空=默认 |
| spu | string | 否 | SPU 关键字过滤（包含匹配，不区分大小写） |

**响应 Body**

```json
{
  "code": 0,
  "data": {
    "store": "1店",
    "storeName": "一店",
    "days": ["2026/2/21","2026/2/22","2026/2/23","2026/2/24","2026/2/25","2026/2/26","2026/2/27"],
    "list": [
      {
        "spu": "AL-W0161",
        "total": [122, 119, 98, 105, 88, 95, 82],
        "diff": -13,
        "pids": [
          {
            "pid": "1729439482671567205",
            "days": [2, 81, 33, 3, 3, 8, 5],
            "diff": -3
          }
        ]
      }
    ]
  }
}
```

**字段说明**

| 字段 | 类型 | 说明 |
|---|---|---|
| store | string | 店铺编码 |
| storeName | string | 店铺展示名 |
| days | string[] | 7 天日期数组（按 endDate 计算） |
| list[].spu | string | SPU 名称 |
| list[].total | int[] | SPU 小计（7 天销量数组） |
| list[].diff | int | 近两天差异（第 7 天 - 第 6 天） |
| list[].pids[].pid | string | PID |
| list[].pids[].days | int[] | 该 PID 7 天销量数组 |
| list[].pids[].diff | int | 该 PID 近两天差异 |

### 4.2 AI 分析（按 SPU）

```
POST /api/bi/tk/sales/trend/ai-analysis
```

**请求 Body**

```json
{
  "store": "1店",
  "spu": "AL-W0161",
  "endDate": "2026-02-27"
}
```

**响应 Body**

```json
{
  "code": 0,
  "data": {
    "spu": "AL-W0161",
    "analysis": "【一店 · AL-W0161】近两日：2026/2/26 合计 95 件，2026/2/27 合计 82 件，环比下降；近两天差异 -13。\n\n共 4 个 PID。\n\n差异为负的 PID：...67205（-3）、...78149（-5）、...75173（-4），建议关注库存或推广。\n\n以上为基于该 SPU 数据的结构化摘要，可配合运营做补货或复盘。"
  }
}
```

### 4.3 导出 CSV

```
GET /api/bi/tk/sales/trend/export
```

查询参数同 4.1，返回 `text/csv`，字段：店铺名、SPU(公式)、PID、7天日期列、近两天差异。

## 五、销售数据映射 需求二：销量趋势数据表

### 5.1 查询列表

```
GET /api/bi/tk/sales/trend-detail
```

**查询参数**

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| store | string | 否 | 店铺（如 `美区跨境1店:US`），空=全部 |
| payDate | string | 否 | 支付日期，格式 `YYYY-MM-DD`；筛选该日支付的订单 |
| page | int | 否 | 页码，默认 1 |
| pageSize | int | 否 | 每页条数，默认 10 |

**响应 Body**

```json
{
  "code": 0,
  "data": {
    "total": 8,
    "page": 1,
    "pageSize": 10,
    "list": [
      {
        "paidTime": "2026-02-27 10:15",
        "month": "202602",
        "store": "美区跨境1店:US",
        "orderId": "ORD-20260227-001",
        "platformSkuId": "1729439482671567205",
        "sellerSku": "AL-W0161-DARKGRAY-XL",
        "skuQty": 2,
        "orderAmount": "57.76",
        "pid": "1729439482671567205",
        "spu": "AL-W0161"
      }
    ]
  }
}
```

**字段说明**

| 字段 | 类型 | 说明 |
|---|---|---|
| paidTime | string | Paid Time（支付时间） |
| month | string | 归属月份(公式)，格式 YYYYMM |
| store | string | 店铺 |
| orderId | string | order id（平台订单编号） |
| platformSkuId | string | Platform SKU ID |
| sellerSku | string | Seller sku（卖家 SKU） |
| skuQty | int | SKU sold quantity（订购数量） |
| orderAmount | string | Order total amount（买家实付金额） |
| pid | string | PID |
| spu | string | SPU(公式) |

### 5.2 导出 CSV

```
GET /api/bi/tk/sales/trend-detail/export
```

查询参数同 5.1（不含分页），返回 `text/csv` 文件流。
